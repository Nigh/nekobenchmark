# NekoBenchmark

一个本地运行的 Godot 4 反应速度测试工具，支持 Windows、Linux 和 macOS。程序以全屏项目菜单启动。

## 使用

从 GitHub Releases 下载与你的系统对应的导出包并运行。界面仅提供英文。

- 菜单：鼠标选择项目；`Esc` 退出。
- 测试内：`Esc` 返回菜单；结果页按 `R` 重试。
- 颜色反应测试：`Space`、`Z`、`X`、方向键或鼠标左键开始和作答。
- 拐角伏击测试：鼠标可在 180° 水平范围内观察场景，敌人在视野内时鼠标左键即可作答。
- OSU：每轮随机出现 6 个不重叠的编号圆圈，按 1–6 顺序点击；反应键推进下一个圆圈。从点中第一个到最后一个计时。点错整轮作废。
- 球形瞄准：随机等待 1–3 秒后同时出现 6 个球体，准星射线命中射击。从出现到打完计时；提前开火或 6 秒内未打完整轮作废；开火间隔至少 150 ms。

颜色反应与拐角伏击连续测量 5 次，随机等待 1–4 秒，目标出现后 1 秒未输入即超时。提前输入、超时，或射击测试中未命中，都会作废整轮并清空样本。完成后显示中位数、平均值和样本标准差。

菜单会显示每个项目的最佳成绩（最低中位数），并保存到 Godot 的 `user://scores.txt`（键：`color`、`shooter`、`osu`、`spheres`），因此重启后仍会保留。

## 拐角伏击测试

该项目将传统反应测试映射为 low-poly 3D 场景：玩家不能移动，但可在 180° 水平范围内转动镜头。敌人在首个目标帧随机出现在左或右掩体，并在一秒内移动至对侧掩体；这个移动周期与响应超时一致。敌人处于相机视锥内即可作答，不要求准星瞄准。

## 界面

界面仅使用 Maple Mono。结果页突出显示中位数、平均值和样本标准差。3D 场景使用 Godot 的 `Camera3D`、`WorldEnvironment`、光源、标准材质、网格和深度缓冲。

Maple Mono 以 SIL Open Font License 1.1 随发布包分发。

## 精度与延迟

### 已采取的措施

- 反应类项目使用独立于 Godot 节点的状态机，确保试次、随机延迟、超时和统计公式一致；序列类项目（OSU / Sphere Aim）使用各自的小状态机。
- 目标状态与 `Time.get_ticks_usec()` 的起始时间在同一 Godot 处理帧设置，且不在该转换中加载资源或创建网格。
- 时间以引擎内微秒差计量。

### 软件无法可靠计算的延迟

- 显示合成器、驱动和显示器可能仍会同步或缓冲帧。
- 显示器扫描方向、像素响应时间与过冲。
- USB 或无线输入设备的固件、轮询率和传输延迟。
- 操作系统输入处理、线程调度及事件队列延迟。

因此该工具适合在相同设备与相同显示配置上做可重复的个人对比，而不是作为跨设备绝对反应时间的校准仪器。

## 本地构建

需要 Godot 4.4 或更高版本（及匹配的导出模板）。Linux/Windows 建议使用 Forward+；Compatibility 是经过测试的后备渲染器。

```sh
godot --headless --path . --script tests/reaction_state_test.gd
godot --headless --path . --script tests/sequence_state_test.gd
godot --headless --path . --script tests/playthrough_test.gd
godot --headless --path . --editor --quit
```

在编辑器中打开项目后，选择对应的导出预设生成发布包。
